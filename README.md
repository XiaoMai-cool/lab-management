## 功能模块

### 界面设计原则

**管理与使用分离，管理按职责分组：**
- 底部导航：首页 | 物资 | 药品 | 采购 | 我的
- 管理面板按职责分组（彩色标题区分）：
  - 采购审批（绿色，教师独立入口）
  - 报销管理（黄色：报销审批、报销统计）
  - 耗材管理（蓝色：申领审批、库存管理、物资追踪、入库登记）
  - 药品管理（紫色：药品库存、补货管理、入库登记）
  - 系统管理（灰色：公告、人员、日常须知、数据导出）

### 登录页（无需登录即可查看）
- 公告通知（管理员可选择哪些公告在登录页展示，点击展开详情）

### 首页 Dashboard
- 待办事项（管理角色看到待审批数量、今日归还提醒等）
- 公告栏（支持图片展示和文件下载，查看全部链接）
- 快捷操作：申领物资、采购申请、药品总览、值日查询、文档资料
- 值日安排（实验室今日+本周排班、办公室本月）
- 日常须知（实验室/办公室分区，从数据库读取，管理端可编辑）
- 库存预警（仅管理角色可见）

### 耗材管理（使用功能）
- 物资总览：四分类，搜索
- 申领物资：**多选申领**，按分类分组展开，搜索过滤，每项设数量
  - 用途选填 + 快捷标签（课题实验、教学用、仪器维护、样品处理、安全防护、其他）
  - 可归还物品由管理员预设 `is_returnable` 属性
- 我的申领：多耗材摘要显示，点进详情
- 归还/报损：归还恢复库存，报损不恢复（中性用语，无追责）

### 药品管理（使用功能）
- 药品总览：120种药品，按编号分区（A/B/C/D），按类型分类（手机端可折叠）
- 药品信息：名称、编号、分子式、CAS号、GHS安全标签
- **取药清单**：搜索药品→加入清单→全屏查看（编号+名称，按编号排序）
  - 可保存清单到数据库（跨设备）、加载已保存清单、比对变更
- **预警优化**：确认弹窗防误触、上报人可撤回（仅 pending 状态）、一个药品只有一条有效预警
- 添加药品：编号自动生成（A/B/C/D 前缀 + 自定义）、名称输入实时显示已有相似药品

### 采购流程（统一表 purchases）
```
学生提交采购申请 → 教师审批 → 学生采购 → 补充报销信息 → 报销审批 → 入库登记
```
- **一条记录走完全流程**（审批→报销→入库）
- 11 个类别：试剂药品、实验耗材、设备配件、服装劳保、测试加工、会议培训、出版知产、办公用品、差旅交通、邮寄物流、其他
- 类别动态字段：试剂药品额外填 CAS号/浓度/纯度等，耗材设备填物品名称/规格
- 采购类型：个人/公共
- 无需入库：系统按类别自动设置，学生不可见，教师审批时可调整
- 教师/板块负责人自动审批通过
- 学生-教师对应关系：自动默认选中，可手动切换
- 附件上传（图片/PDF/Word/Excel）：申请时可上传发票，有发票+金额自动进入报销
- 报销前可修改金额和附件
- 报销金额差异 ≥50 元提醒
- 入库登记：耗材专人看非药品，药品专人看试剂药品，可编辑/删除

### 撤回功能
- 教师撤回采购审批
- 报销审批人撤回报销
- 耗材审批撤回（库存回滚）
- 管理员撤销归还（库存回滚）
- 药品补货撤回送达

### 报销统计
- 6 个 Tab：全部、试剂药品、实验耗材、服务费用、行政开支、其他
- 筛选：姓名、时间范围、金额范围、类别、采购类型（手机端可折叠）
- 汇总卡片 + 分类统计 + 导出 Excel

### 公告与文档管理（三合一面板）
- **公告管理**：富文本编辑（TipTap），附件上传（图片/PDF/Word/Excel），可选登录页展示
- **文档管理**：富文本编辑，附件上传，PDF 导出（浏览器打印），按分类分组
- **日常须知**：实验室/办公室分区，可增删改排序
- 富文本工具栏：加粗、斜体、H2/H3 标题、有序/无序列表、链接、图片插入
- 旧纯文本内容自动兼容（无需数据迁移）
- 文档附件：图片可展开预览（查看者决定），其他文件直接下载
- 超级管理员可查看存储用量（进度条 + 80% 预警）

### 其他功能
- 值日排班：实验室每人一天/4周轮换 + 办公室每月一换（待重构为管理端可配置）
- 数据导出：耗材库存、预约记录、药品记录、采购记录（Excel 格式）

## 数据备份与恢复

系统已配置自动数据库备份，所有数据会定期导出为 JSON 文件，保存在仓库的 `backups/` 文件夹中。

### 自动备份（无需操作）

- **频率**：每周一、周四早上 9:00（北京时间）自动运行
- **内容**：导出全部 25 张表的数据
- **保留**：最近 10 次备份，旧备份自动清理
- **存储**：自动提交到 GitHub 仓库 `backups/` 目录

### 手动触发备份

如果想立刻备份（不用等自动执行），按以下步骤操作：

1. 打开 GitHub 仓库页面：https://github.com/XiaoMai-cool/lab-management
2. 点击顶部的 **Actions** 标签（在 Code / Issues / Pull requests 右边）
3. 左侧列表点击 **定期数据库备份**
4. 右侧点击 **Run workflow** 按钮
5. 在弹出的下拉框中点击绿色的 **Run workflow**
6. 等待约 30 秒，刷新页面，看到绿色 ✅ 就表示备份完成
7. 回到 Code 标签 → 打开 `backups/` 文件夹，可以看到最新的备份

### 恢复数据（出问题时用）

如果数据出了问题需要恢复，联系开发者运行以下命令：

```bash
# 查看可用的备份
ls backups/

# 恢复到指定日期的备份（例如 2026-04-05 的备份）
SUPABASE_SERVICE_KEY=你的key node scripts/restore-data.mjs backups/2026-04-05_145431
```

> ⚠️ 恢复会用备份数据覆盖当前数据库，请谨慎操作。

### 备份文件说明

每次备份会生成一个按日期时间命名的文件夹（如 `2026-04-05_145431`），里面包含：
- 每张表一个 `.json` 文件（如 `chemicals.json`、`supplies.json`）
- `_summary.json`：本次备份的摘要（表数量、记录数、备份时间）
