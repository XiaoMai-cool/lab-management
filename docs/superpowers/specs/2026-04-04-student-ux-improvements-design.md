# 学生使用端体验改进设计

## 背景

学生在使用耗材申领和药品查询功能时存在多个体验痛点：耗材申领每次只能选一个、申领和借用是两个独立流程但逻辑重复、药品查找需要手动抄写编号到纸上、药品预警容易误触且无法撤回。

## 改动范围

1. **耗材申领改进** — 合并申领+借用、多选、分类展开、用途选填
2. **药品使用改进** — 取药清单、预警交互优化

---

## 一、耗材申领改进

### 现状

- `SupplyReserve.tsx`（申领）：需审批，有「使用后归还」勾选框，每次只能选一个耗材
- `SupplyBorrow.tsx`（借用）：不需审批，直接扣库存，默认要归还
- 两个流程逻辑重复，学生不清楚该用哪个
- 用途为必填项
- 耗材列表在下拉菜单中，数量多时不好找

### 目标

合并申领和借用为统一流程，支持多选，简化操作。

### 流程设计

统一流程：

```
学生申领 → 老师审批 → 审批通过（扣库存）→ 根据物品属性：
  - 消耗品 → 结束
  - 可归还物品 → 使用中 → 归还 或 报损
```

#### 物品属性

在耗材库存管理（`SupplyManage.tsx`）中，每个耗材新增 `is_returnable` 属性（布尔值），由管理员预设：
- `true`：可归还物品（烧杯、插座等）
- `false`：消耗品（手套、枪头等）

学生申领时能看到物品的可归还标记，但不需要自己选择。

#### 多选申领

申领页面改为表单内直接展示耗材列表：
- 按分类分组展开/收起
- 搜索功能保留，实时过滤
- 每个耗材可勾选，勾选后显示数量输入框
- 已选耗材汇总显示在底部
- 一次提交所有选中的耗材，老师一次审批

#### 审批方式

一次提交生成一条审批记录，包含多个耗材明细。老师审批时看到完整清单：
- 默认整体通过/驳回
- 老师可以删掉个别项（不批准的耗材）
- 老师可以调整个别项的数量
- 一键提交审批结果
- 学生看到结果如"3项申领，2项通过，1项被移除"

#### 用途改为选填

- 非必填
- 提供快捷标签按钮，点击自动填入：
  - 课题实验
  - 教学用
  - 仪器维护
  - 样品处理
  - 安全防护
  - 其他（选择后聚焦到手动输入框）
- 也可不选标签，直接手动输入自定义内容

#### 归还与报损

可归还物品审批通过后进入「使用中」状态，学生在归还页面操作：
- **归还**：库存加回来
- **报损**：库存不加回来，记录损耗原因（选填），属于正常操作不追责

### 页面变更

| 当前页面 | 变更 |
|---------|------|
| `SupplyReserve.tsx` | 改为多选表单，按分类分组展示耗材，移除「使用后归还」勾选框，用途改选填+快捷标签 |
| `SupplyBorrow.tsx` | 废弃，功能合并到申领流程 |
| `SupplyReturn.tsx` | 保留，「丢失」改为「报损」，去除追责暗示 |
| `SupplyList.tsx` | 保留，SubNav 中移除「借用」入口 |
| `SupplyManage.tsx`（管理端） | 每个耗材增加 `is_returnable` 属性配置 |
| `ReservationReview.tsx`（管理端） | 适配多耗材审批，支持整体通过/驳回 + 逐个删除/调整数量 |

### 数据模型变更

`supplies` 表新增字段：

```
is_returnable -- boolean, 是否可归还，默认 false
```

`supply_reservations` 表改为支持多耗材：

新增 `supply_reservation_items` 表：

```
supply_reservation_items:
  id
  reservation_id    -- 关联 supply_reservations
  supply_id         -- 耗材 ID
  quantity          -- 申领数量
  approved_quantity -- 审批通过数量（老师可调整，null 表示未审批）
  removed           -- boolean, 老师审批时删除此项则为 true
```

`supply_reservations` 表调整：
- 移除 `supply_id`、`quantity` 字段（移到 items 表）
- 移除 `is_returnable` 字段（改从 supplies 表读取）
- 保留 `purpose`（改为选填）、`status`、`reviewer_id` 等

`supply_borrowings` 表保留用于追踪可归还物品的使用状态：
- 审批通过后，可归还物品自动创建 borrowing 记录
- 状态：`borrowed`（使用中）→ `returned`（已归还）/ `damaged`（已报损）
- 原有 `lost` 状态改为 `damaged`（报损），旧数据为测试数据无需迁移

### 底部导航调整

「物资」Tab 进入后的 SubNav 移除「借用」入口，保留：物资列表、申领、我的申领、归还。

## 二、药品使用改进

### 现状

- 学生需要逐个搜索药品，手动抄写编号到纸上
- 预警按钮无确认环节，容易误触
- 预警上报后无法撤回
- 已有预警时其他人仍能看到预警按钮

### 2.1 取药清单

#### 功能描述

在药品列表页（`ReagentList.tsx`）增加「取药清单」功能：

- 每个药品卡片增加「加入清单」按钮
- 点击后药品加入浮动清单
- 清单显示在页面底部（浮动条），展示已添加数量，点击展开完整清单
- 清单内容：编号、药品名称、存放位置、规格
- 可从清单中移除单个药品
- 清单操作：
  - **清空**：一键清除临时清单
  - **保存**：命名后保存（如"XX实验常用药品"），下次可复用

#### 已保存清单

- 保存的清单存在数据库中，跨设备可用
- 在药品列表页可快捷加载已保存清单
- 可删除已保存清单
- 可编辑已保存清单（重命名、增删药品）

#### 数据存储

新增 `reagent_lists` 表：

```
reagent_lists:
  id
  user_id           -- 所属用户
  name              -- 清单名称
  items             -- jsonb, 药品列表 [{ chemical_id, name, batch_number, location, specification }]
  created_at
  updated_at
```

RLS 策略：用户只能读写自己的清单。

### 2.2 预警交互优化

#### 确认环节

点击「预警」按钮后弹出确认框：
- 标题：确认上报「XX药品」即将用完？
- 说明文字：上报后所有人将看到「即将用完」标签
- 按钮：取消 / 确认上报

#### 上报人撤回

- 上报人在药品详情或列表中看到自己上报的预警时，显示「撤回」按钮
- 点击撤回后弹确认框
- 撤回后「即将用完」标签消失

#### 预警唯一性

- 同一药品同时只有一条有效预警（status = 'pending' 或 'ordered'）
- 已有有效预警时，所有用户隐藏「预警」按钮
- 所有用户都能看到「即将用完」标签

### 页面变更

| 当前页面 | 变更 |
|---------|------|
| `ReagentList.tsx` | 增加「加入清单」按钮、浮动清单栏、已保存清单加载入口、预警按钮加确认框、已有预警时隐藏按钮、上报人显示撤回按钮 |
| `ReagentDetail.tsx` | 预警按钮加确认框、上报人显示撤回按钮 |

### 数据模型变更

`chemical_warnings` 表：无需变更，现有字段已支持（`reported_by` 字段可用于判断上报人）。

新增 `reagent_lists` 表（见上方数据存储）。

## 不在范围内

- 耗材库存的自动补货/采购联动
- 药品用量追踪（记录每次取了多少）
- 通知系统（预警推送等）
- 旧数据迁移（现有数据为测试数据）
