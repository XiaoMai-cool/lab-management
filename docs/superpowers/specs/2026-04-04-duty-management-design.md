# 值班管理重构设计

## 背景

当前值班管理完全硬编码在代码中（Dashboard.tsx 和 DutyRoster.tsx），人员名单、轮换规则、起始日期都写死。管理员无法通过界面修改排班，加减人员或调整顺序需要改代码。

## 目标

将值班管理改为数据库驱动，管理员可通过界面配置规则、调整人员、手动覆盖排班。

---

## 值班类型

### 实验室值日
- 5人（或更多），分配周一到周五，每人负责一天
- 每 N 周轮换一次（默认4周），整体往后挪一天（周一的人变周二，周五的人变周一）
- 周末无值日

### 办公室值日
- 同一批人，但顺序可以不同
- 每月一人负责，按顺序轮换

---

## 管理页面

入口：系统管理分组 → 值日管理（duty 模块权限可访问）

### 实验室值日设置

**人员列表：**
- 显示当前顺序的人员列表
- 操作：添加人员、删除人员、上下移动调整顺序、插入到指定位置
- 添加人员时可选择生效时间：立即生效 / 从指定日期开始

**轮换设置：**
- 轮换周期：每 N 周轮一次（数字输入，默认 4）
- 起始参考日期：日期选择器（系统从这天开始计算轮换）

**手动覆盖：**
- 选择某一周（日期选择器选周一）
- 直接指定周一到周五分别谁值班
- 覆盖为永久生效，之后的排班从覆盖点重新计算

### 办公室值日设置

**人员列表：**
- 同实验室，可独立排序
- 顺序可以跟实验室不同

**轮换设置：**
- 轮换周期：每 N 月轮一次（默认 1）
- 起始参考月份

**手动覆盖：**
- 选择某个月
- 指定谁负责该月
- 永久生效

## 显示逻辑

### Dashboard 首页
- 今日值日：根据规则自动计算今天谁值日（周末显示"今日无值日"）
- 本周排班：显示周一到周五的值日人员
- 办公室本月：显示本月负责人
- 如果有手动覆盖，用覆盖的人员

### 值日排班页面（/duty）
- 显示与 Dashboard 相同的信息
- 管理员额外看到"编辑排班"按钮进入管理页面

### 登录页
- 不显示值日信息（之前讨论过不需要）

## 数据模型

### duty_config 表（值班规则配置）

```
duty_config:
  id
  type                -- 'lab' | 'office'
  people              -- text[], 有序人员列表，如 ['陈鸿琳', '麦宏博', '彭鸿昌', '邓岩昊', '林弋杰']
  rotation_period     -- integer, 轮换周期（实验室为周数，办公室为月数）
  ref_date            -- date, 起始参考日期
  created_at
  updated_at
```

每种类型只有一条记录（lab 一条，office 一条）。

### duty_overrides 表（手动覆盖）

```
duty_overrides:
  id
  type                -- 'lab' | 'office'
  target_date         -- date, 实验室为该周的周一日期，办公室为该月1号
  people              -- text[], 实验室为5人[周一到周五]，办公室为1人[负责人]
  created_at
```

### RLS 策略

- 所有登录用户可读（SELECT）
- 值日管理员（manages_module('duty')）和管理员可写（INSERT/UPDATE/DELETE）

## 计算逻辑

### 实验室值日计算

1. 从 `duty_config` 读取 lab 配置：people, rotation_period, ref_date
2. 检查 `duty_overrides` 是否有匹配本周的覆盖记录
3. 如果有覆盖：直接用覆盖的 people 数组
4. 如果没有覆盖：
   - 计算从 ref_date 到本周一的周数
   - 除以 rotation_period 取余，得到当前轮换偏移量
   - 将 people 数组按偏移量循环移位
   - 周一到周五分别对应 people[0] 到 people[4]

### 办公室值日计算

1. 从 `duty_config` 读取 office 配置：people, rotation_period, ref_date
2. 检查 `duty_overrides` 是否有匹配本月的覆盖记录
3. 如果有覆盖：直接用覆盖的 people[0]
4. 如果没有覆盖：
   - 计算从 ref_date 所在月到当前月的月数
   - 除以 (people.length * rotation_period) 取余
   - 对应 people 数组中的某个人

## 页面变更

| 页面 | 变更 |
|-----|------|
| 新增：`DutyManage.tsx` | 值日管理页面（规则配置 + 手动覆盖） |

| `App.tsx` | 添加 `/admin/duty-manage` 路由 |

---

## 旧的 duty_roster 表

现有的 `duty_roster` 表用于手动添加排班记录，新方案用 `duty_config` + `duty_overrides` 替代。旧表数据为测试数据，不需要迁移。

---

## 不在范围内

- 调班/换班功能（口头交接即可）
- 值日完成打卡/确认
- 通知提醒
